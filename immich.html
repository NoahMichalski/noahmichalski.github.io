Immich VM Setup & Update Guide

Proxmox + Ubuntu + Docker Deployment

Part 1 — Creating the Immich VM

1. Create VM in Proxmox

  • Allocate 8GB+ RAM
  • Allocate 4 CPU cores
  • Set CPU type to host
  • Create disk (1TB+ recommended)
  • Install Ubuntu Server

2. Install Docker

sudo apt update
sudo apt install ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu noble stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker $USER

3. Deploy Immich

mkdir ~/immich
cd ~/immich

git clone https://github.com/immich-app/immich.git
cd immich/docker

cp example.env .env

docker compose up -d

Access Immich at: http://VM_IP:2283

4. Expanding Disk Later

sudo growpart /dev/sda 3
sudo pvresize /dev/sda3
sudo lvextend -l +100%FREE /dev/ubuntu-vg/ubuntu-lv
sudo resize2fs /dev/ubuntu-vg/ubuntu-lv

Part 2 — Updating Immich

1. Navigate to Immich Folder

cd ~/immich/immich/docker

2. Pull Latest Images

docker compose pull

3. Restart With Updated Containers

docker compose up -d

4. Optional: Clean Old Images

docker image prune -f

5. Verify Containers

docker ps

Recommended Maintenance

  • Back up Immich database regularly
  • Back up library directory separately
  • Keep VM RAM at 8GB+
  • Monitor disk usage with: df -h
