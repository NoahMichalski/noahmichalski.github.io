2Ubuntu Setup

After installing Ubuntu, SSH into the VM and update it.

ssh noah@192.168.1.191
sudo apt update && sudo apt upgrade -y
sudo reboot

After reboot, reconnect and install Java 17 plus basic tools.

sudo apt install -y openjdk-17-jre-headless wget curl screen unzip nano
java -version

The server showed Java 17 correctly:

openjdk version "17.0.19"

3Create the Server Folder and Install Forge

Create a dedicated minecraft system user and server directory.

sudo adduser --system --home /opt/minecraft --group minecraft
sudo mkdir -p /opt/minecraft
sudo chown -R minecraft:minecraft /opt/minecraft

Give your user access to work in the folder:

sudo usermod -aG minecraft noah
sudo chmod -R 775 /opt/minecraft

Log out and back in after adding yourself to the group.

exit
ssh noah@192.168.1.191

Download Forge Installer

cd /opt/minecraft
wget -O forge-installer.jar https://maven.minecraftforge.net/net/minecraftforge/forge/1.20.1-47.4.20/forge-1.20.1-47.4.20-installer.jar

Install Forge Server

java -jar forge-installer.jar --installServer

After install, the folder should contain:

libraries/
run.sh
user_jvm_args.txt
forge-installer.jar

Fix Ownership

sudo chown -R minecraft:minecraft /opt/minecraft
sudo chmod -R 775 /opt/minecraft

Set Server RAM

Edit:

sudo nano /opt/minecraft/user_jvm_args.txt

Add this at the bottom:

-Xms4G
-Xmx8G

4First Launch, EULA, and Mods

First Launch

Run once to generate files and the EULA.

cd /opt/minecraft
sudo -u minecraft ./run.sh nogui

It stops with the EULA message. Edit the EULA file:

sudo nano /opt/minecraft/eula.txt

Change:

eula=false

to:

eula=true

Test Forge Without Mods

cd /opt/minecraft
sudo -u minecraft ./run.sh nogui

Wait for:

Done (...)! For help, type "help"

Then stop it cleanly:

stop

Create Mods Folder

cd /opt/minecraft
sudo -u minecraft mkdir -p mods

Upload Mods from Windows

The mod files were stored on Windows at:

\\arctic\h\ArcticPC\DownloadsMini

PowerShell upload commands:

cd "\\arctic\h\ArcticPC\DownloadsMini"

6systemd Service / Auto Start

Create the service file:

sudo nano /etc/systemd/system/minecraft.service

Paste:

[Unit]
Description=Minecraft Forge 1.20.1 Server
After=network.target

[Service]
WorkingDirectory=/opt/minecraft
User=minecraft
Group=minecraft
Type=simple
Restart=on-failure
RestartSec=10

ExecStart=/bin/bash /opt/minecraft/run.sh nogui

StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target

Enable and start:

sudo systemctl daemon-reload
sudo systemctl enable minecraft
sudo systemctl start minecraft

Check status:

sudo systemctl status minecraft

Confirm it loaded:

sudo journalctl -u minecraft -n 80 --no-pager

We confirmed:

Done (6.436s)! For help, type "help"

8Main PC Client Setup Without CurseForge Launcher

Install Java 17 on the main PC. The issue we hit was that PowerShell still showed Java 8:

java version "1.8.0_491"

We installed Java 17 and launched Forge directly with Java 17.

Important: Running the Forge installer from the network path caused this error: URI has an authority component. The fix was copying the Forge installer to the local Downloads folder first.

Copy Forge Installer Local

Copy-Item "\\arctic\h\ArcticPC\DownloadsMini\forge-1.20.1-47.4.20-installer.jar" "$env:USERPROFILE\Downloads\forge-1.20.1-47.4.20-installer.jar"

Run Forge Installer with Java 17

cd "$env:USERPROFILE\Downloads"
& "C:\Program Files\Java\jdk-17\bin\java.exe" -jar ".\forge-1.20.1-47.4.20-installer.jar"

Choose Install client. Then open the normal Minecraft Launcher and select:

forge 1.20.1-forge-47.4.20

Launch Forge once, quit Minecraft, then open the client mods folder:

explorer "$env:APPDATA\.minecraft\mods"

Place these two mod files in the client mods folder:

geckolib-forge-1.20.1-4.8.3.jar
the_wonderland-2.1.4-1.20.1.jar

Join locally with:

192.168.1.191:25565

10OP / Moderator Setup

Once you are OP, you can OP friends inside the game:

/op FriendUsername

To remove OP:

/deop FriendUsername

If nobody is OP yet, stop the service and start the server manually once:

sudo systemctl stop minecraft
cd /opt/minecraft
sudo -u minecraft ./run.sh nogui

Wait for Done, then type in the server console:

op YourMinecraftUsername

Then stop and restart the service:

stop
sudo systemctl start minecraft
Do not start two copies of the server. If the service is running and you start manually, Minecraft will throw a session.lock error.

Manual Backup Before Changing Mods

Stop the server, backup the world, then start the server again.

sudo systemctl stop minecraft
sudo mkdir -p /opt/minecraft-backups
sudo tar -czvf /opt/minecraft-backups/world-$(date +%F-%H%M).tar.gz -C /opt/minecraft world
sudo systemctl start minecraft

List backups:

ls -lah /opt/minecraft-backups

Troubleshooting Notes We Hit

Java on Windows Still Shows Java 8

PowerShell showed:

java version "1.8.0_491"

The workaround was launching Forge with Java 17 directly:

& "C:\Program Files\Java\jdk-17\bin\java.exe" -jar ".\forge-1.20.1-47.4.20-installer.jar"

Forge Installer Error from Network Path

java.lang.IllegalArgumentException: URI has an authority component

Fix: copy the installer to local Downloads before running it.

Wrong GeckoLib File

Do not use:

geckolib-neoforge-26.1.2-5.5.1.jar

Use the Forge 1.20.1 file:

geckolib-forge-1.20.1-4.8.3.jar

session.lock Error

Error:

/opt/minecraft/./world/session.lock: already locked

Cause: the Minecraft service was already running, and another manual server was started.

Fix:

sudo systemctl stop minecraft
sudo systemctl status minecraft

Then start manually only if needed:

cd /opt/minecraft
sudo -u minecraft ./run.sh nogui
