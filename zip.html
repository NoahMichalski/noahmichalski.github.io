Zipline VM Setup + Update Guide

A quick repeatable guide for rebuilding or updating Noah’s Zipline server on Proxmox using Ubuntu Server, Docker Compose, PostgreSQL, and Nginx Proxy Manager.

Current Working Setup

VM Resources 2 vCPU 4 GB RAM 500 GB disk
Operating System Ubuntu Server
Zipline URL https://zip.arcticmatrix.com
Install Path /opt/zipline
Upload Storage /opt/zipline/uploads
Reverse Proxy Nginx Proxy Manager → HTTP backend on port 3000
Important Do not lose the .env file. It contains the database password and CORE_SECRET. Back up the whole /opt/zipline folder before major updates.

Fresh Install Steps

Use these steps if you ever rebuild the VM from scratch.

1. Update Ubuntu

sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget git nano htop qemu-guest-agent ca-certificates
sudo systemctl enable --now qemu-guest-agent

2. Install Docker + Compose

sudo apt update
sudo apt install -y ca-certificates curl

sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

sudo tee /etc/apt/sources.list.d/docker.sources > /dev/null <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

3. Enable Docker and test it

sudo systemctl enable --now docker
sudo docker run hello-world

4. Let your user run Docker without sudo

sudo usermod -aG docker $USER
newgrp docker
docker ps

If docker ps works without a permission error, Docker is ready.

Zipline Folder + Docker Compose

1. Create the Zipline folder

sudo mkdir -p /opt/zipline
sudo chown -R $USER:$USER /opt/zipline
cd /opt/zipline

mkdir -p uploads public themes postgres

2. Generate secrets

Run these and save the outputs somewhere safe.

openssl rand -hex 32
openssl rand -hex 24

First one = CORE_SECRET. Second one = PostgreSQL password.

3. Create .env

Replace the example values with your generated values.

nano .env
POSTGRESQL_USER=zipline
POSTGRESQL_PASSWORD=PASTE_POSTGRES_PASSWORD_HERE
POSTGRESQL_DB=zipline

CORE_SECRET=PASTE_CORE_SECRET_HERE

4. Create docker-compose.yml

nano docker-compose.yml
services:
  zipline:
    image: ghcr.io/diced/zipline:latest
    container_name: zipline
    restart: unless-stopped
    ports:
      - "3000:3000"
    environment:
      - CORE_SECRET=${CORE_SECRET}
      - DATABASE_URL=postgres://${POSTGRESQL_USER}:${POSTGRESQL_PASSWORD}@postgres:5432/${POSTGRESQL_DB}
      - DATASOURCE_TYPE=local
      - DATASOURCE_LOCAL_DIRECTORY=/zipline/uploads
    volumes:
      - ./uploads:/zipline/uploads
      - ./public:/zipline/public
      - ./themes:/zipline/themes
    depends_on:
      - postgres

  postgres:
    image: postgres:16
    container_name: zipline-postgres
    restart: unless-stopped
    environment:
      - POSTGRES_USER=${POSTGRESQL_USER}
      - POSTGRES_PASSWORD=${POSTGRESQL_PASSWORD}
      - POSTGRES_DB=${POSTGRESQL_DB}
    volumes:
      - ./postgres:/var/lib/postgresql/data

5. Start Zipline

cd /opt/zipline
docker compose up -d
docker ps

6. Check logs

docker logs zipline --tail=100

You want to see the server start on port 3000 without errors.

7. First browser setup

  1. Open http://VM-IP:3000.
  2. Create your super-admin user.
  3. Upload a small test file.
  4. Confirm it appears in /opt/zipline/uploads.
ls -lah /opt/zipline/uploads

Reverse Proxy Setup

This is the setup that fixed the 502 issue.

Nginx Proxy Manager host settings
  • Domain Name: zip.arcticmatrix.com
  • Scheme: http
  • Forward Hostname / IP: Zipline VM IP
  • Forward Port: 3000
  • SSL: Request/enable certificate for zip.arcticmatrix.com

Zipline Core Settings

  1. Go to Administrator → Settings → Core.
  2. Set Default Domain to zip.arcticmatrix.com.
  3. Turn on Return HTTPS URLs.
  4. Turn on Trust Proxies.
  5. Save settings.

Optional upload size fix

If large uploads fail through the domain, check Zipline’s own file limit first. If it still fails, add this in Nginx Proxy Manager → Advanced:

client_max_body_size 5G;

How to Update Zipline

Before updating Make a backup first. Do not skip this if you care about your uploaded files and database.

1. Go to the Zipline folder

cd /opt/zipline

2. Quick backup

mkdir -p ~/zipline-backups
tar -czf ~/zipline-backups/zipline-backup-$(date +%F-%H%M).tar.gz /opt/zipline

3. Pull the newest image

docker compose pull

4. Recreate containers

docker compose up -d

5. Confirm it worked

docker ps
docker logs zipline --tail=100

Then open https://zip.arcticmatrix.com and test an upload.

One-line update command

Use this only after you are comfortable with the process:

cd /opt/zipline && docker compose pull && docker compose up -d && docker logs zipline --tail=80

Troubleshooting

502 Bad Gateway / openresty Usually the reverse proxy cannot reach Zipline. In Nginx Proxy Manager, make sure:
  • Scheme is http, not https.
  • Forward IP is the Zipline VM IP.
  • Forward port is 3000.
  • Zipline container is running.
hostname -I
docker ps
docker logs zipline --tail=100
Uploads stop around 100 MB Check Zipline first:
  1. Go to Administrator → Settings → Files.
  2. Raise the max upload size.
  3. Save and test again.
If it still fails, add client_max_body_size 5G; to Nginx Proxy Manager → Advanced.
Links are showing HTTP instead of HTTPS In Zipline:
  1. Go to Administrator → Settings → Core.
  2. Default Domain should be zip.arcticmatrix.com.
  3. Enable Return HTTPS URLs.
  4. Enable Trust Proxies.
Postgres will not start Check permissions and logs: 
cd /opt/zipline
docker logs zipline-postgres --tail=100
ls -lah /opt/zipline/postgres
