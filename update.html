Docker Media Server Update Guide

Quick commands for updating Noah's self-hosted media apps: Audiobookshelf, Navidrome, and FileBrowser.

Audiobookshelf

Compose v2

Use this on the Audiobookshelf VM. You already installed Compose v2 there.

cd ~/audiobookshelf

docker compose pull
docker compose up -d
docker ps

Navidrome + FileBrowser

Compose v2 preferred

Use this after the Navidrome VM has Docker Compose v2 installed.

cd ~/navidrome

docker compose pull
docker compose up -d
docker ps

Old Compose v1 fallback

If the VM only understands docker-compose with a dash, use this. This fixed your Navidrome/FileBrowser ContainerConfig issue last time.

cd ~/navidrome

docker-compose pull
docker-compose down --remove-orphans
docker-compose up -d
docker ps
Never run: docker-compose down --volumes unless you are intentionally deleting named volumes.

Audiobookshelf Update

Port 13378 → 80

Your Audiobookshelf container uses the image ghcr.io/advplyr/audiobookshelf:latest and is reached at port 13378.

  1. SSH into the Audiobookshelf VM

    ssh noah@AUDIOBOOKSHELF_IP

  2. Go to the Compose folder

    cd ~/audiobookshelf
ls

    Make sure you see docker-compose.yml or compose.yml.

  3. Pull the newest image and restart

    docker compose pull
docker compose up -d

  4. Confirm it is running

    docker ps

    Look for audiobookshelf and port 13378->80.

  5. Optional cleanup

    docker image prune -f

Navidrome + FileBrowser Update

Watch for old Compose v1

Your Navidrome VM previously used the legacy docker-compose command. After installing Compose v2, use docker compose.

Preferred Compose v2 method

cd ~/navidrome

docker compose pull
docker compose up -d
docker ps

If Compose v2 is not installed yet

docker compose version

If that says docker: unknown command: docker compose, the VM does not have Compose v2 yet.

Legacy fallback

cd ~/navidrome

docker-compose pull
docker-compose down --remove-orphans
docker-compose up -d
docker ps
Why the down/up step? Old Compose v1 can crash with KeyError: 'ContainerConfig' during updates. Running down --remove-orphans and then up -d recreates the containers cleanly without deleting your mounted data.

Install Docker Compose v2 on a VM

Use this when a VM only has the old docker-compose command or says it cannot find docker compose.

sudo apt update
sudo apt install ca-certificates curl gnupg -y

sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

sudo chmod a+r /etc/apt/keyrings/docker.gpg

. /etc/os-release

echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$VERSION_CODENAME stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install docker-compose-plugin -y

docker compose version

Troubleshooting

Problem What it means Fix
docker: unknown command: docker compose Compose v2 is not installed. Install docker-compose-plugin or use legacy docker-compose.
Unable to locate package docker-compose-plugin The Docker official apt repo is not added yet. Use the Compose v2 install block above.
KeyError: 'ContainerConfig' Old Compose v1 bug with newer images/Docker metadata. Use Compose v2, or run docker-compose down --remove-orphans then docker-compose up -d.
No containers show in docker ps Containers are stopped or failed to recreate. Run docker ps -a, then run Compose up -d from the right folder.
App looks reset/fresh Wrong volume paths may be mounted. Stop and check your compose file volume paths before using the app heavily.

Useful checks

# Show running containers
docker ps

# Show all containers, even stopped
docker ps -a

# Check Compose v2
docker compose version

# Check old Compose v1
docker-compose version

# View logs for a service
docker compose logs -f SERVICE_NAME
Safety rule: Avoid down --volumes, docker volume prune, or deleting folders under your media/config paths unless you are 100% sure you have backups.
