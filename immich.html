2) Install Docker (and the exact fixes we hit)

We hit multiple Docker install problems: unsupported Ubuntu release repo mismatch, and a broken GPG keyring format that caused NO_PUBKEY errors. This section is the clean, repeatable method.
Clean install (Docker + Compose plugin)Ubuntu

sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor --yes -o /etc/apt/keyrings/docker.gpg
sudo chmod 644 /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu noble stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
Fix: docker.sock permission deniedAdd user to docker group

sudo usermod -aG docker $USER

# IMPORTANT: log out and back in (or exit SSH and reconnect)

groups
docker ps

3) Install Immich (how we actually got it working)

We tried release downloads for .env and got tiny files or 404: Not Found. The reliable solution was cloning Immich and using immich/docker where the real compose files live.
Clone Immich + start the stackRepo method (reliable)

sudo apt install -y git

mkdir -p ~/immich
cd ~/immich

git clone https://github.com/immich-app/immich.git
cd immich/docker

# Immich uses example.env
cp example.env .env

docker compose up -d

docker ps

4) Problems we hit (why + fixes)

Docker packages not foundRepo mismatch
Your Ubuntu release didn’t have Docker packages. Fix: add Docker’s repo for a supported release (we used noble).
GPG key ignored / NO_PUBKEYKeyring wrong format
Fix: regenerate /etc/apt/keyrings/docker.gpg using gpg --dearmor and set perms to 644.
.env file empty or “404: Not Found”Bad download URL
Fix: clone the repo and copy example.env to .env.
Docker permission deniedUser not applied
Fix: add user to docker group and reconnect your shell session.

5) Expand disk (1TB → 2TB) inside Ubuntu (LVM)

After resizing in Proxmox, Ubuntu showed sda as 2T but sda3 stayed 998G. We expanded the partition, then LVM, then filesystem.
A) Resize disk in ProxmoxUI step
VM → Hardware → select disk → Resize Disk → enter increment in GiB (example: add 1TB = 1024 GiB).
B) Confirm new disk sizeInside VM

lsblk
C) Expand partition 3 to fill diskgrowpart

sudo apt install -y cloud-guest-utils
sudo growpart /dev/sda 3
D) Expand LVM + filesystempvresize → lvextend → resize2fs

sudo pvresize /dev/sda3
sudo lvextend -l +100%FREE /dev/ubuntu-vg/ubuntu-lv
sudo resize2fs /dev/ubuntu-vg/ubuntu-lv

df -h

6) How to update Immich (walkthrough)

Updating is simple: pull new images, then recreate containers. Your data stays in volumes and your upload library.
1) Go to the compose folderImportant: correct path

cd ~/immich/immich/docker
2) Pull new imagesDownloads updates

docker compose pull
3) Apply updatesRecreate containers

docker compose up -d
4) Verify healthHealthy containers

docker ps

docker compose logs --tail=80
If Immich ever fails after an update, grab logs: docker compose logs --tail=200.
